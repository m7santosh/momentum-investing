# NSE Monthly Seasonality Analysis

Seasonality analysis is now available through a single Tkinter UI launcher.

## Directory Structure

```
seasonality/
├── README.md                    # This file
├── seasonality_simple.py        # Simplified screening analysis
├── seasonality_backtest.py      # Comprehensive backtest analysis
├── seasonality_ui.py            # GUI launcher and results viewer
└── bat/                         # Windows batch files
    └── seasonality_ui.bat       # Launch the seasonality UI
```

## Quick Start

### Run the GUI
```batch
seasonality\bat\seasonality_ui.bat
```

### Workflow
- Use the **Screen** tab to inspect seasonality for selected indices or all major indices.
- Use the **Backtest** tab to run a full index backtest and see summary output in the same window.

### Direct Python command
```bash
python seasonality/seasonality_ui.py
```

## Notes
- The UI captures analysis output directly inside the window.
- There is no need to run individual console scripts manually.
- The batch file simply launches the GUI.

### Option 3: Python Command Line

From the seasonality folder:

```bash
# Demo
python seasonality_demo.py

# Quick analysis
python seasonality_quick.py

# Simple analysis (with options)
python seasonality_simple.py
python seasonality_simple.py --years 5
python seasonality_simple.py --years 5 --indices "Nifty 50" "Nifty Bank"

# Comprehensive analysis (with options)
python seasonality_backtest.py
python seasonality_backtest.py --years 5 --top-n 20
```

## Analysis Types

| Script | Indices | Time | Best For |
|--------|---------|------|----------|
| **demo** | 1 (Nifty 50) | ~30 sec | Learning, validation |

| **backtest** | 120+ all | ~15-30 min | Research, comprehensive view |

## Output Metrics

Each analysis produces these metrics for each month:

- **Avg Return %**: Average monthly return across all years
- **Win Rate %**: Percentage of years the month was positive
- **Avg Gain %**: Average return when the month was positive
- **Avg Loss %**: Average return when the month was negative
- **Sample Count**: Number of years with data for that month

## Output Files

- Location: `final_result/seasonality_monthly_simple.csv`
- Contains: Detailed metrics for each index and month
- Generated by: `seasonality_simple.py` and `seasonality_backtest.py`

### Console Output
All scripts output results to the console with:
- Monthly seasonality table
- Win rates by month
- Best and worst performing months
- Index-wise summary (for comprehensive analysis)

## Parameters

### Common Options

**--years N**
- Number of years to analyze (default: 10)
- Example: `--years 5` for last 5 years

**--top-n M** (backtest only)
- Show top M indices by average monthly return
- Example: `--top-n 20` for top 20 indices

**--indices INDEX1 INDEX2 ...** (simple only)
- Analyze specific indices
- Example: `--indices "Nifty 50" "Nifty Bank" "Nifty IT"`

## Data Sources

- **Primary**: Yahoo Finance (NSEIDX tickers)
- **Fallback**: NSE EOD (bhavcopy)
- **Cache**: Local NSE data under `data/nse/` folder
- **Leverages**: Existing `utils.india_market_data` infrastructure

## Integration with Backtest Framework

These scripts integrate with the existing momentum investing backtesting framework:

- Uses `utils.india_market_data.get_india_market_data()` for data loading
- Uses `momentum.index.nifty_indices` for index definitions
- Respects local NSE data caching strategy
- Exports results to `final_result/` directory

## Requirements

## Performance Notes

- First run: Downloads and caches NSE data (~5-10 min for comprehensive analysis)
- Subsequent runs: Uses cached data (~2-5 min for comprehensive analysis)
- Demo: Always fast (~30 sec) as it only uses Nifty 50

## Troubleshooting

### "No data available"
- Ensure internet connection for first run
- Check that NSE is accessible
- Try with shorter time period: `--years 3`

### Slow performance
- First run downloads NSE data; subsequent runs are faster
- Run `seasonality_quick.bat` instead for faster analysis

### Missing indices
- Some indices have limited history
- Scripts automatically skip indices with insufficient data
- Use `--years 3` or less for newer indices
